# 커서(Cursor)에서 개발 시작하기

이 폴더에는 성심당 스탬프 웹 데모 개발에 필요한 기획 문서 2개가 들어 있습니다.

| 파일 | 내용 |
|---|---|
| PRD.md | 제품 요구사항 문서 (화면별 요구사항, 디자인 가이드, 기술 스택) |
| USER_FLOW.md | 사용자 흐름 문서 (플로우별 단계, 예외 처리, 상태별 화면) |

---

## 시작 방법

1. 커서에서 새 프로젝트 폴더를 만들고, 이 두 파일을 프로젝트 안에 `docs` 폴더로 넣습니다.

```
내-프로젝트/
└── docs/
    ├── PRD.md
    └── USER_FLOW.md
```

2. 커서 채팅(Agent)에 아래 프롬프트를 그대로 붙여넣어 시작합니다.

---

## 시작 프롬프트 (복사해서 사용)

```
docs/PRD.md와 docs/USER_FLOW.md를 먼저 정독하고 개발을 시작해줘.

프로젝트 조건:
- Next.js + Tailwind CSS로 새 프로젝트 셋업 (Vercel 배포 전제)
- 백엔드·Firebase 사용 금지, 모든 데이터는 localStorage에 저장
- 모바일 웹 우선 (375~430px), 데스크톱에서는 모바일 폭 중앙 정렬

구현 순서:
1. 프로젝트 셋업 + 디자인 토큰 (PRD 4장 디자인 가이드의 크림 배경,
   성심당 레드, 브라운 컬러를 Tailwind 커스텀 컬러로 정의)
2. 최초 방문 닉네임 설정 플로우 (USER_FLOW 1장)
3. 메인 화면 — 스탬프 판, 최근 방문 지점, 하단 내비게이션 (PRD 3.1)
4. 스탬프 적립 시뮬레이션 + 도장 찍기 애니메이션 팝업 (PRD 3.2, USER_FLOW 2장)
5. 스탬프 내역 달력 화면 (PRD 3.3, USER_FLOW 3장)
6. 마이페이지 닉네임 변경 (PRD 3.4, USER_FLOW 4장)

품질 기준:
- 임시 디자인, 기본 템플릿 룩 금지. PRD 4장의 디자인 가이드
  (1956 헤리티지, 수채화 감성, 도장 느낌 스탬프)를 반드시 반영
- 에러가 나면 나에게 묻지 말고 스스로 진단하고 해결한 뒤,
  무엇이 문제였고 어떻게 고쳤는지 한 문장으로만 알려줘

각 단계가 끝날 때마다 화면을 확인할 수 있게 알려줘.
```

## Vercel 배포 (개발 완료 후)

1. 코드를 GitHub 저장소에 올립니다 (커서에게 "GitHub에 올려줘"라고 요청 가능)
2. vercel.com 접속 → GitHub 계정으로 로그인 → "Add New Project"
3. 방금 올린 저장소 선택 → 설정 변경 없이 "Deploy" 클릭
4. 1~2분 후 배포 완료, 접속 주소가 발급됩니다
5. 이후 GitHub에 코드를 올릴 때마다 자동으로 재배포됩니다
